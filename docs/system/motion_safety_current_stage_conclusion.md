# Motion Safety 当前阶段研究结论

## 当前阶段目标

当前阶段的目标已经明确收敛为：

- 不再以“泛化跌倒检测”为目标
- 重点研究运行期间的未离台危险状态
- 重点区分：
  - 正常使用
  - 危险异常倒伏 / 危险塌陷 / 危险失姿

换句话说，当前阶段研究的是：

> 用户仍未完全离台，但已经进入危险姿态，而设备持续振动可能造成二次伤害。

## 当前观察到的行为空间

### 正常使用类

当前正常使用类样本仍然是关键 hard negatives，包括但不限于：

- 正常律动
- 正常姿态波动
- 非危险的受力迁移
- 可恢复的姿态变化

这些样本的重要性在于：

- 它们与危险状态共享部分运动/承重变化特征
- 如果 hard negatives 不足，规则很容易把正常使用误判为危险状态

### 危险状态类

当前阶段关注的危险状态类，不是所有 fall-like 行为，而是：

- 运行中出现危险倒伏
- 身体仍部分留台或仍与平台支撑相关
- 持续振动可能带来二次伤害

## `FALL_ON_PLATFORM` 的结构化理解

当前研究已经明确：

- `FALL_ON_PLATFORM` 不是单一结构

至少应区分为两类：

### A型：支撑型危险倒伏

定义：

- 用户在平台或平台相关支撑面上发生明显塌陷
- 危险状态主要仍由平台承重或主要支撑体现

特征理解：

- 更容易表现出明显低承重 / 持续低承重
- 更接近“深低承重危险状态”

### B型：外部支撑分担型危险倒伏

定义：

- 用户已进入危险倒伏状态
- 但部分重量被外部支撑分担
- 例如上半身被凳子、手支撑、地面接触或其他外部支撑结构部分承接

特征理解：

- 仍是危险状态
- 但未必出现特别深的低承重尾段
- 可能表现为：
  - 明显迁移
  - 恢复不足
  - 但尾段承重不一定极低

## 关键研究发现

### 发现 1：当前主问题不是 fully off-platform fall

fully off-platform 情况可能已经部分落入 leave-platform 处理边界。

当前阶段主要补的是：

- 尚未完全离台
- 但已经危险

这一部分空白。

### 发现 2：危险状态不能只由深低承重定义

如果仅用“深低承重”定义危险状态，会丢掉 B 型：

- 外部支撑分担型危险倒伏

这意味着：

- 深低承重仍然是重要路径
- 但不能作为唯一危险定义

### 发现 3：B 型样本改变了规则演进方向

新采集的 `FALL_ON_PLATFORM` 样本表明：

- 危险状态不一定意味着极低尾段承重
- “异常迁移后未恢复”路径必须被强化

这也是 v0.3 被接受为当前研究规则工作版的重要原因。

## 为什么危险状态不能只定义为 deep low-load

原因很明确：

1. A 型可以更容易表现为 deep low-load
2. B 型则可能被外部支撑部分托住
3. 如果只看深低承重，就会把部分真实危险状态排除掉

因此当前阶段的正式认识是：

> 危险状态定义必须同时容纳 A 型与 B 型，不能把“深低承重”当作唯一判断语义。

## 当前阶段接受结论

当前阶段可以正式接受以下结论：

1. 当前主目标是未离台危险状态
2. 当前重点不是 generic fall detection
3. `FALL_ON_PLATFORM` 至少包含 A/B 两种结构
4. B 型是当前研究中必须保留的重要危险子类
5. 正常使用样本仍是关键 hard negatives
6. v0.3 可作为当前研究规则工作版

## 当前阶段仍未解决的问题

当前阶段尚未解决：

- 危险状态与 leave-platform 的细粒度边界分离
- runtime-ready 的稳定 detector 设计
- production-ready 的参数与验证闭环

因此当前不能声称：

- 已完成 runtime 规则
- 已完成 firmware 集成标准
- 已具备生产可用的危险状态检测能力

## 下一阶段建议

下一阶段应基于当前书面基线继续推进，但应保持边界清晰：

### 应继续做

- 用当前阶段结论作为正式参考口径
- 在 Demo APP / 研究流程中继续支持样本复核、标注、导出
- 继续针对 danger vs leave-platform 边界采样与验证

### 不应立即做

- 直接把 v0.3 宣布为正式 runtime 规则
- 直接写入 firmware 自动决策
- 把当前研究版描述为最终检测方案

## 当前阶段正式口径

当前阶段的正式口径应写为：

> SonicWave 当前研究重点是“未离台危险状态”而非泛化跌倒检测。`FALL_ON_PLATFORM` 已确认至少包含 A型：支撑型危险倒伏 与 B型：外部支撑分担型危险倒伏。当前接受的研究规则工作版为 v0.3，但它仍未完成 danger vs leave-platform 的细粒度分离，因此不能视为正式 runtime 规则。
